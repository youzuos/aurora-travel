# Aurora v3.0 用户完整路径流程

Updated: 2026-06-30

用途：记录当前确认中的 v3.0 用户使用整体逻辑，方便产品、设计、前端、Agent 逻辑共同对齐。此文件是可编辑文本版本，后续修改应同步更新 PRD 或 PRD changelog。

```text
用户首次打开 Aurora
│
├─ 判断：用户是否已有年度计划？
│   │
│   └─ 系统先问用户：
│       「今年你有没有一定要去旅行的地方？」
│
├─ 用户回答：有
│   │
│   ├─ 进入「年度心愿填写」
│   │   │
│   │   ├─ 用户填写一定要去的地方 / 想看的风景
│   │   │   ├─ 具体目的地：京都 / 冰岛 / 新疆 / 巴黎
│   │   │   ├─ 具体体验：樱花 / 极光 / 胡杨 / 雪山 / 鲸鱼
│   │   │   └─ 模糊心愿：想去海边 / 想看雪 / 想吃美食
│   │   │
│   │   ├─ 系统将输入转成「年度心愿夹」
│   │   │
│   │   ├─ 用户给每个心愿设置优先级
│   │   │   ├─ 必去
│   │   │   ├─ 想去
│   │   │   └─ 随缘
│   │   │
│   │   └─ 用户继续补充边界条件
│   │       ├─ 年假天数
│   │       ├─ 年度旅行预算
│   │       ├─ 计划旅行次数
│   │       ├─ 单次旅行平均预算
│   │       └─ 不能旅行的具体日期
│   │
│   └─ 进入「Aurora Agent 生成年度计划」
│
├─ 用户回答：没有 / 还没想好
│   │
│   ├─ 进入「灵感发现首页」
│   │   │
│   │   ├─ 页面目标：
│   │   │   └─ 派小精灵去为用户寻找世界上最值得去旅行一次的地方
│   │   │
│   │   ├─ 用户拥有 5 次探索机会
│   │   │
│   │   ├─ 每次探索前，用户可以选择：
│   │   │   ├─ 输入具体城市：京都 / 巴黎 / 冰岛
│   │   │   ├─ 输入国家地区：日本 / 北欧 / 新疆
│   │   │   ├─ 输入模糊偏好：海边 / 雪山 / 美食 / 小众古城
│   │   │   └─ 不输入，随机派小精灵出发
│   │   │
│   │   ├─ 小精灵出发探索
│   │   │
│   │   ├─ 返回「旅行见闻卡片」
│   │   │   ├─ 城市 / 国家
│   │   │   ├─ 小精灵见闻
│   │   │   ├─ 图片 / 明信片
│   │   │   ├─ 标签：樱花 / 极光 / 美食 / 雪山等
│   │   │   └─ 操作按钮：
│   │   │       ├─ 加入年度心愿夹
│   │   │       ├─ 换个方向
│   │   │       └─ 继续附近探索
│   │   │
│   │   ├─ 判断：用户是否在 5 次内选择了目的地？
│   │   │   │
│   │   │   ├─ 是
│   │   │   │   └─ 将选择的目的地加入「年度心愿夹」
│   │   │   │
│   │   │   └─ 否
│   │   │       └─ 系统随机指定一个目的地加入「年度心愿夹」
│   │   │
│   │   └─ 进入「年度心愿夹」
│   │
│   ├─ 年度心愿夹
│   │   │
│   │   ├─ 心愿来源包括：
│   │   │   ├─ 用户主动输入的目的地
│   │   │   ├─ 小精灵推荐后加入的目的地
│   │   │   └─ 5 次探索后系统随机指定的目的地
│   │   │
│   │   ├─ 用户给每个心愿设置优先级：
│   │   │   ├─ 必去
│   │   │   ├─ 想去
│   │   │   └─ 随缘
│   │   │
│   │   └─ 用户继续补充边界条件
│   │       ├─ 年假天数
│   │       ├─ 年度旅行预算
│   │       ├─ 计划旅行次数
│   │       ├─ 单次旅行平均预算
│   │       └─ 不能旅行的具体日期
│   │
│   └─ 进入「Aurora Agent 生成年度计划」
│
├─ Aurora Agent 生成年度计划
│   │
│   ├─ 时机 Agent
│   │   └─ 分析花期 / 极光 / 雪季 / 胡杨 / 天气窗口
│   │
│   ├─ 假期 Agent
│   │   └─ 分析法定节假日 / 年假拼接 / 不可出行日期
│   │
│   ├─ 综合 Agent
│   │   ├─ 按优先级排序心愿
│   │   ├─ 计算年假是否够用
│   │   ├─ 计算预算是否够用
│   │   ├─ 判断哪些目的地可以拼成一次旅行
│   │   └─ 判断哪些心愿需要推迟到下一年
│   │
│   └─ 提醒小助手
│       └─ 生成后续提醒节点：
│           ├─ 订票
│           ├─ 天气
│           ├─ 装备
│           └─ 预约
│
├─ 生成「Year View 年度计划」
│   │
│   ├─ 展示年度甘特图
│   │   ├─ 每段旅行在哪个月
│   │   ├─ 每段旅行当前成熟度
│   │   │   ├─ Sketch
│   │   │   ├─ Refining
│   │   │   └─ Ready
│   │   ├─ 被推迟到下一年的心愿
│   │   └─ 不可出行日期 / 冲突提示
│   │
│   ├─ 展示年度概览
│   │   ├─ 年假使用情况
│   │   ├─ 预算使用情况
│   │   ├─ 心愿覆盖情况
│   │   └─ 计划旅行次数
│   │
│   ├─ 展示小精灵最近见闻
│   │   ├─ 小精灵继续探索目的地
│   │   ├─ 推荐新地点 / 新体验
│   │   └─ 用户可以加入年度心愿夹
│   │
│   ├─ 用户可以修改计划
│   │   ├─ 增加新心愿
│   │   ├─ 修改心愿优先级
│   │   ├─ 修改预算 / 年假 / 出行次数
│   │   └─ 修改不可出行日期
│   │
│   └─ 用户点击某段旅行
│
├─ Trip View 单次旅行精修
│   │
│   ├─ 展示这次旅行的核心信息
│   │   ├─ 目的地组合
│   │   ├─ 推荐出行日期
│   │   ├─ 行程天数
│   │   ├─ 需要请几天年假
│   │   └─ 使用了哪些公共假期
│   │
│   ├─ 展示请假策略
│   │   ├─ 方案 1：峰值优先
│   │   ├─ 方案 2：提前出发
│   │   └─ 方案 3：假期拼接
│   │
│   ├─ 展示体验时机
│   │   ├─ 花期 / 极光 / 雪季概率
│   │   ├─ 当前扑空率
│   │   ├─ 推荐峰值窗口
│   │   └─ 备选窗口
│   │
│   ├─ 展示本次行程心愿夹
│   │   ├─ 已加入的具体地点 / 餐厅 / 景点
│   │   ├─ 小精灵推荐的新见闻
│   │   └─ 用户可以加入本次行程
│   │
│   ├─ 展示天粒度行程
│   │   ├─ Day 1 去哪里
│   │   ├─ Day 2 去哪里
│   │   ├─ Day 3 去哪里
│   │   ├─ 下雨 / 闭馆 / 天气变化时的备选方案
│   │   └─ 如果目的地太少、剩余天数较空：
│   │       ├─ 用话术提示用户可以继续增加目的地
│   │       ├─ 推荐小精灵继续探路
│   │       └─ 或标记为「随缘逛逛 / 自由探索时间」
│   │
│   └─ 用户返回 Year View
│
├─ Time Warp 时间推进
│   │
│   ├─ 年初
│   │   ├─ 计划状态：Sketch
│   │   ├─ 信息粒度：概率区间 / 大致季节 / 灵感层
│   │   └─ 小精灵见闻：为什么这个地方值得去
│   │
│   ├─ 出发前 3 月
│   │   ├─ 计划状态：Refining
│   │   ├─ 信息粒度：推荐月份 / 大致预算 / 组合路线
│   │   └─ 小精灵见闻：附近还能拼哪些地方
│   │
│   ├─ 出发前 1 月
│   │   ├─ 计划状态：Refining → Ready
│   │   ├─ 信息粒度：街区 / 景点 / 餐食 / 照片点
│   │   └─ 小精灵见闻：具体去哪、怎么安排更顺
│   │
│   └─ T-14
│       ├─ 计划状态：Ready
│       ├─ 信息粒度：天气 / 预约 / 交通 / 打包 / 备选方案
│       └─ 小精灵见闻：临近出发提醒
│
├─ 临近提醒
│   │
│   ├─ T-45
│   │   └─ 提醒：可以关注机票 / 酒店 / 签证 / 请假审批
│   │
│   ├─ T-14
│   │   └─ 提醒：天气变化 / 景点开放 / 预约 / 交通
│   │
│   ├─ T-3
│   │   └─ 提醒：装备清单 / 备用路线 / 最终确认
│   │
│   └─ 出发当天
│       └─ 用户带着 Aurora 生成的个人化计划出发
│
└─ 全年循环
    │
    ├─ 小精灵继续探索
    ├─ 用户继续加入新心愿
    ├─ Aurora 重新评估年度计划
    ├─ 无法达成的心愿推迟到下一年
    └─ 每段旅行随着时间逐渐从 Sketch → Refining → Ready
```
