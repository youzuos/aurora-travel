# 旅行 Agent 功能总结

更新时间：2026-06-30

## 1. 功能定位

旅行 Agent 是 Aurora 项目中的拟人化旅行陪伴功能。它会以一个可爱像素风小动物的形式出现，代替用户在不同城市中旅行、观察、聊天、拍照和分享见闻。

该功能的核心目标是让旅行规划工具不只是静态表单和计划生成器，而是具备陪伴感、持续互动感和故事感。用户可以随时询问 Agent 当前在哪里、在做什么、看到了什么，也可以指定它前往某个城市或国家。

## 2. 已实现功能

### 2.1 首次选择旅伴形象

用户首次使用时可以选择一个旅行 Agent 形象。当前已提供 8 个像素风小动物旅伴：

- 小青蛙
- 小狐狸
- 小猫
- 小狗
- 小兔
- 小熊
- 小企鹅
- 小熊猫

用户选择后，形象会保存在本地状态中。后续用户也可以通过“换角色”重新选择旅伴。

### 2.2 像素风小动物形象

每个旅伴都采用像素风小动物设计，并带有旅行元素：

- 背包
- 相机
- 围巾
- 不同身体姿态
- 不同情绪状态

当前支持的状态和动作包括：

- 普通待机
- 行走
- 拍照
- 吃东西
- 看地图
- 睡觉
- 兴奋

这些状态会根据当前语境自动变化。例如用户要求它“发照片”时，小动物会切换为拍照状态；当地时间较晚时，它会进入睡觉或安静状态。

### 2.3 首页旅伴近况

在开屏首页，旅行 Agent 会展示当前所在城市和当地时间，并用拟人化语气告诉用户它现在正在那里经历什么。

示例：

> 我刚躲进一条安静的小巷，木窗缝里有一点茶香飘出来。

该信息会根据 Agent 当前城市、时间和状态变化。

### 2.4 悬浮 Agent

进入主界面后，旅行 Agent 会以悬浮卡片的形式固定在屏幕角落。悬浮卡片展示：

- 当前小动物形象
- 当前城市
- 当前状态

用户点击悬浮卡片后，可以打开聊天面板，与 Agent 直接对话。

### 2.5 主动聊天

旅行 Agent 会周期性主动发送近况，不会长期沉默。主动消息内容包括：

- 当前城市见闻
- 遇到的人
- 看到的风景
- 吃到的小吃
- 旅行中的小事件

自动迁移和主动发言的节奏已经调整，避免过于频繁。测试模式下可以加快节奏，方便开发和展示前验证。

### 2.6 用户即时聊天

用户可以随时向旅行 Agent 发消息。Agent 会立即先给出本地回复，随后在配置了 LLM API 的情况下用模型增强回复内容。

支持的典型问题包括：

- 你在干嘛？
- 你在哪里？
- 你吃了什么？
- 发一张照片
- 讲讲风景
- 遇到什么人了吗？

Agent 的回复会尽量展开话题，而不是只做简单问答。

### 2.7 指定旅行目的地

用户可以通过聊天指定 Agent 前往某个城市、国家或类型目的地。

示例：

- 去巴黎
- 去冰岛
- 去海边
- 去一个安静有寺庙的地方

Agent 会根据输入匹配目的地，并更新：

- 当前城市
- 当前国家
- 当地时间
- 旅行状态
- 聊天记录

当用户指定具体城市时，会主动触发迁移。

### 2.8 自动迁移逻辑

旅行 Agent 会在一个城市停留一段时间后自动迁移到下一个城市。自动迁移优先选择相邻或相关城市，避免跳转过于随机。

正式使用中迁移间隔较长，不会频繁切换城市。测试模式中迁移时间可以缩短，用于交付前快速验证。

### 2.9 城市当地时间匹配

Agent 的行为会根据当前所在城市的当地时间变化。

已支持的时间逻辑：

- 夜间会进入睡觉或安静状态
- 白天更倾向于行走、拍照、看风景
- 正餐时间可以触发食物相关状态
- 小吃不严格限制在正餐时间，只要 Agent 醒着就可以吃

例如 Agent 在京都时，会使用京都当地时间判断状态；迁移到巴黎后，会自动使用巴黎当地时间。

### 2.10 图片功能

用户要求 Agent 发照片时，Agent 会返回城市照片卡片。

当前方案不是每次调用 AI 生成图片，而是使用预设城市照片池。这样更稳定、更省 token，也更适合课程展示。

图片消息包括：

- 城市照片
- 图片说明
- 图片来源说明

示例：

> 这张京都有一点晨雾，连屋檐都像在轻声说话。

### 2.11 语音功能

Agent 可以发送语音类消息。当前语音使用浏览器内置的 `speechSynthesis` 播放，不消耗 LLM API token。

语音消息会在聊天面板中显示播放按钮，例如 `8s`。用户点击后即可播放对应文本。

### 2.12 Gemini API 接入

当前已接入 Google Gemini 文本 API，用于增强旅行 Agent 的自然语言回复。

后端调用优先级为：

1. Gemini
2. Azure OpenAI
3. OpenAI
4. 本地 fallback

也就是说，如果配置了 Gemini API key，会优先使用 Gemini；如果没有配置，系统仍然可以使用本地规则回复，不会导致功能崩溃。

需要在 `.env.local` 中配置：

```env
GEMINI_API_KEY=你的GoogleAIStudioKey
GEMINI_MODEL=gemini-2.5-flash-lite
```

### 2.13 本地 fallback

即使没有任何 LLM API key，旅行 Agent 仍然可以正常运行。

本地 fallback 支持：

- 即时聊天
- 发照片
- 发语音类消息
- 指定城市迁移
- 自动迁移
- 当地时间状态判断
- 换角色

区别是：没有 LLM 时，回复会更模板化；接入 Gemini 后，回复会更自然、更像真人。

## 3. 已测试功能

当前已完成以下验证：

- 回归测试通过：`npm run test:companion`
- 生产构建通过：`npm run build`
- 浏览器实测首页可打开
- 浏览器实测悬浮 Agent 可点击
- 浏览器实测聊天面板可打开
- 浏览器实测用户发送消息后 Agent 会回复
- 浏览器实测“发照片”会返回城市照片卡片
- 浏览器实测“去巴黎”会触发迁移
- 浏览器实测迁移后城市和当地时间会更新
- 浏览器实测“换角色”可以重新选择小动物
- 浏览器实测语音按钮可以点击
- API fallback 实测无 key 时仍返回本地回复
- Gemini provider 已通过 mock API 回归测试

## 4. 当前实现边界

以下功能目前没有作为主功能实现：

- 用户上传自己的图片生成专属 Agent：暂不考虑，作为进阶功能保留说明
- 每次实时生成旅行照片：暂不默认启用，避免消耗大量 token 和影响稳定性
- 真正的音频文件生成：当前使用浏览器语音播放，成本更低
- 大规模真实网络爬取旅行照片：当前使用城市照片池，避免版权、稳定性和加载风险

## 5. 当前方案优势

当前旅行 Agent 的实现比较适合 7 月 4 日前交付展示：

- 没有 API key 也能运行
- 接入 Gemini 后可以增强回复质量
- 图片和语音成本低
- 功能覆盖聊天、照片、语音、迁移、换角色、当地时间状态
- 视觉上有小动物形象和动态状态
- 交互上比普通旅行规划工具更有陪伴感

## 6. 后续可增强方向

后续如果还有时间，可以继续增强：

- 为每个城市增加更多照片和见闻素材
- 增加“旅行日记”时间线，记录 Agent 去过哪里
- 增加“明信片收藏”，保存 Agent 发过的照片
- 增加“心情值/疲惫值”，让 Agent 更像长期陪伴角色
- 增加更自然的主动消息频率控制
- 增加用户上传图片生成专属小动物形象
- 增加真正的 Gemini 图片生成，但只在用户明确要求时触发
